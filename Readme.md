# QuadXtreme — Backend (часткова реалізація)

Backend-частина студентського проєкту QuadXtreme — платформи для бронювання екстрим-подорожей на квадроциклах.

ВАЖЛИВО:
Даний репозиторій містить лише серверну (Back-end) частину проєкту.
Front-end (мобільний застосунок на React Native) буде реалізований на наступних етапах розробки.

----------------------------------------------------------------

МЕТА ПРОЄКТУ

Метою проєкту є розробка серверної частини мобільного застосунку, яка забезпечує:
- роботу з базою даних;
- REST API для взаємодії з клієнтською частиною;
- авторизацію та автентифікацію користувачів;
- бізнес-логіку бронювання екстрим-турів;
- обмін повідомленнями між користувачем і гідом.

----------------------------------------------------------------

РЕАЛІЗОВАНИЙ ФУНКЦІОНАЛ (BACKEND)

На поточному етапі реалізовано:
- реєстрацію користувачів;
- авторизацію користувачів з використанням JWT;
- ролі користувачів (user / guide / admin);
- перегляд списку турів;
- створення турів (для авторизованих користувачів);
- бронювання турів;
- перегляд власних бронювань;
- REST-чат між користувачем та гідом у межах бронювання;
- валідацію вхідних даних;
- захист API-маршрутів middleware-ами.

----------------------------------------------------------------

ТЕХНОЛОГІЧНИЙ СТЕК

Backend:
- Node.js (v16)
- Express.js
- PostgreSQL 17
- Sequelize ORM
- JWT (jsonwebtoken)
- bcryptjs
- Joi (валідація)
- dotenv

Архітектура:
- клієнт-серверна;
- REST API;
- модульна структура коду.

----------------------------------------------------------------


БАЗА ДАНИХ

Використовується PostgreSQL 17.

Основні таблиці:
- users
- tours
- bookings
- messages

UUID генеруються:
- або на стороні PostgreSQL через gen_random_uuid() (pgcrypto),
- або на стороні Sequelize.

ENUM-и в PostgreSQL не використовуються, щоб уникнути конфліктів із Sequelize.

----------------------------------------------------------------

НАЛАШТУВАННЯ СЕРЕДОВИЩА

Файл .env має містити:
PORT=4000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=quadxtremedb
DB_USER=postgres
DB_PASSWORD=your_password
JWT_SECRET=super_secret_key

----------------------------------------------------------------

ЗАПУСК ПРОЄКТУ

Встановлення залежностей:
npm install

Запуск сервера:
node src/server.js

Успішний запуск супроводжується повідомленнями:
QuadXtreme backend is running on port 4000
Database connection established
Models synchronized with database

----------------------------------------------------------------

ОСНОВНІ API ЕНДПОІНТИ

Авторизація:
POST /api/auth/register
POST /api/auth/login

Тури:
GET /api/tours
GET /api/tours/:id
POST /api/tours (потрібен JWT)

Бронювання:
POST /api/bookings (потрібен JWT)
GET /api/bookings/me (потрібен JWT)

Чат:
POST /api/chat (потрібен JWT)
GET /api/chat/booking/:bookingId (потрібен JWT)

----------------------------------------------------------------

АВТОРИЗАЦІЯ

Для доступу до захищених маршрутів необхідно передавати заголовок:
Authorization: Bearer JWT_TOKEN

----------------------------------------------------------------

ПОДАЛЬШИЙ РОЗВИТОК

На наступних етапах планується:
- розробка мобільного застосунку (React Native);
- інтеграція з картографічними сервісами;
- інтеграція платіжних систем;
- розширення функціоналу для гідів та адміністраторів;
- оптимізація та тестування.

----------------------------------------------------------------

ПРИЗНАЧЕННЯ ПРОЄКТУ

Проєкт створено в навчальних цілях як приклад побудови серверної частини сучасного мобільного застосунку з використанням актуального стеку технологій.
